# Configuration Reference

This document describes runtime configuration surfaces and defaults.

## Configuration Layers

1. **Environment variables**
2. **Persisted settings in SQLite (`settings` table)**
3. **Filesystem-backed data under `user_data/`**

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `XPDITE_USER_DATA_DIR` | `<project_root>/user_data` (dev) | Root for DB, memories, artifacts, screenshots, and skills |
| `XPDITE_RUNTIME_ROOT` | unset | Packaged runtime root used by Electron to point Python at bundled `source/` and `mcp_servers/` assets |
| `XPDITE_RUNTIME_ENV_FILE` | unset | Explicit packaged env file path for allowlisted runtime secrets such as Google OAuth desktop credentials |
| `XPDITE_CHILD_PYTHON_EXECUTABLE` | unset | Packaged Python interpreter used when the backend launches Python-based MCP subprocess servers |
| `XPDITE_SERVER_HOST` | `127.0.0.1` | Python API bind host |
| `XPDITE_SERVER_TOKEN` | generated by Electron at startup | Session token required by sensitive loopback APIs |
| `XPDITE_THREAD_POOL_SIZE` | `4` | App thread pool size for blocking tasks |
| `PYTHON_SERVER_PORT` | `8000` (auto-select in `8000-8009`) | Port hint used by bridge integration |
| `BRIDGE_PORT` | `9000` | Channel Bridge HTTP service port |
| `XPDITE_MOBILE_DEBUG_LOGS` | unset | Enables verbose mobile bridge debug logging when set to `1` |

Bridge binding note:

- Channel Bridge is expected to run on loopback interfaces for local-only access.

## SQLite Settings Keys

The `settings` table stores user and runtime preferences.

Common keys:

- `enabled_models`
- `api_key_anthropic`, `api_key_openai`, `api_key_gemini`, `api_key_openrouter`
- `encryption_salt`
- `tool_always_on`, `tool_retriever_top_k`
- `sub_agent_tier_fast`, `sub_agent_tier_smart`
- `memory_profile_auto_inject`
- `system_prompt_template`
- `mobile_channel_telegram`, `mobile_channel_discord`, `mobile_channel_whatsapp`

## User Data Layout

`XPDITE_USER_DATA_DIR` contains:

- `xpdite_app.db`
- `screenshots/`
- `artifacts/`
- `memory/`
- `skills/builtin/`
- `skills/user/`
- `skills/preferences.json`
- `mobile_channels_config.json`
- `google/token.json` (after OAuth)

## Google OAuth Packaging

- Development loads `.env` from the repository root.
- Packaged builds load the explicit env file passed by Electron through `XPDITE_RUNTIME_ENV_FILE`.
- The packaged env file is generated at build time and only contains `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET`.
- Missing packaged Google credentials now fail clearly during startup instead of falling back to placeholder values.

## Secret Storage

- Cloud model provider keys are encrypted at rest in SQLite settings.
- Hugging Face speaker-diarization access is no longer read from `HF_TOKEN`; users enter it in Settings > Meeting and it is stored through the encrypted key-manager path under the `huggingface` provider.

## Security-Relevant Config

- Artifact endpoints require loopback host and session token (`X-Xpdite-Server-Token`) when token is present.
- API keys are encrypted at rest before storage.
- Packaged Google OAuth secrets are allowlisted into a dedicated runtime env file instead of bundling the entire repo `.env`.
- Renderer is isolated via Electron security defaults (`contextIsolation`, no renderer Node integration).

## Operational Notes

- Electron sets `XPDITE_USER_DATA_DIR` in packaged production builds.
- In development, backend falls back to local project `user_data/`.
- Channel Bridge configuration is generated from SQLite settings and synced to `mobile_channels_config.json`.
